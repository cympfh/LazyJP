## 3. プラグイン側の仕様（Neovim）

### 3.1 役割
- ユーザーの入力を監視
- Ctrl + m などのトリガーでエンジンに非同期で送信
- 結果が返ってきたらバッファを更新
- ユーザーが編集した場合は変換をキャンセル

### 3.2 基本動作フロー

1. ユーザーが1行を書く
2. **Ctrl + m** を押す
   - 行が空 → 何も送信せず改行のみ
   - 行に内容あり → エンジンに非同期送信 + 改行挿入
3. 送信した行を「変換待ちリスト」に追加（行番号＋ハッシュで管理）
4. エンジンから結果が返ってきたら：
   - リストに残っていれば → 元のローマ字行を削除して日本語に置換
   - すでに編集されていたら → 無視（キャンセル）

### 3.3 変換待ちリスト管理
- プラグイン側でリストを保持
- `TextChanged` などのautocmdで監視
- 内容が変更されたらリストから削除 → 後から結果が来ても無視

### 3.4 エンジンとの通信
- HTTPで `/convert` エンドポイントを非同期で呼び出す
- 送信時に以下の情報を送る：
  - 現在行のテキスト
  - 直前2つの変換済み結果（コンテキスト）
  - 文体・言語情報

### 3.5 キャッシュの使い方（プラグイン視点）
- プラグインはキャッシュの存在を意識する必要はない
- エンジンが自動でキャッシュを利用して高速に返す
- 同じ入力を繰り返した場合でも、プラグイン側は特に何もする必要がない

### 3.6 トリガーキー
- デフォルトは `Ctrl + m`
- ユーザーが `init.lua` などで自由に `vim.keymap.set` して変更可能
- プラグイン側に「トリガーキー設定」UIは持たない

### 3.7 エラー時の挙動
- エンジンからエラーが返ってきた場合、`:echo` で一時的にメッセージを表示
- その行はローマ字のまま残す

## 補足
- プラグインはできるだけ薄く保つ
- 複雑なロジック（キャッシュ、LLM呼び出し）はすべてエンジン側に寄せる
- 将来的に他のエディタ（VSCodeなど）への移植も考慮しやすい構造にする
